# 项目介绍
这是一个设备状态监控软件Better Monitor的Android端agent，主要作用是抓取设备信息并上报服务器

## 项目运行逻辑
本项目目前有一个交互API文档（API_DOC.md）,里面介绍了agent和上游服务器的交互方式，具体能实现的功能等，你需要参照这一实现完成项目。

## 核心思路
1. 项目要实现的功能是抓取设备信息并上报服务器，因此需要一个服务来实现这个功能，这个服务需要在后台运行，不能因为用户关闭app界面而停止
2. 可以在通知栏上显示一个服务正在运行的通知，通知内容可以是"Better Monitor Agent is running"
3. 如果你有需求无法完成或者Android API使用有困难，可以找我讨论，如果Android无法实现某个功能，可以找我商量其他方案
4. 默认该程序在无root权限的手机上运行，如果哪个功能需要root权限才能完成而不能通过Android API实现，可以找我商量其他方案
5. 要兼容有root权限的手机，如果真的有root权限，那么可以使用root权限来实现，但一定要先做好权限检查，避免无谓的错误

## 目前项目的一些细节
1. 项目采用kotlin语言，编译配置文件也采用build.gradle.kts
2. 项目Min SDK采用Android11，Target SDK采用Android15，使用各种Android API之前要注意这个版本要求，包括权限管理等
3. 本项目要拿各种各样的参数，文件等，因此编写代码时候一定要注意请求权限和检查权限
4. 项目要有相对完善的log方便我查看状态和调试，log内容最好可以复制出来方便我查找
